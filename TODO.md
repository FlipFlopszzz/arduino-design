## 一、代码优化

### 1. 单播加 ACK 确认（必须等对方确认才打印发送成功）
当前 `D` 命令只发不收，不管对方收没收到就直接打印成功。改成：

```
发 D2Hello
  → 发单播包
  → 等 500ms 对方回复 ACK
  → 收到 ACK → 打印 [TX] >2: Hello (ACK)
  → 超时未收 → 打印 [TX] >2: Hello (NOT ACKED)
```

相关文件：`proj/main/Node.h`，找 `sendUnicast()` 和 `F_DATA` / `F_DATA_ACK`。

## 三、LabVIEW 上位机（需要手动做）

在 LabVIEW 中搭串口通信界面，通过与 Arduino 的 USB 串口通信显示网络状态。

### 串口通信协议

PC → Arduino 命令（已实现）：

| 命令 | 说明 |
|------|------|
| `S` | 返回路由表 + 统计数据 |
| `D<addr><data>` | 发送单播数据 |
| `B<data>` | 发送广播 |
| `T<addr><n>` | 网络质量测试 |
| `L` | 本机退网 |
| `R` | 重置统计 |

### LabVIEW 串口命令发送方法
LabVIEW 使用 **NI-VISA** 驱动进行串口通信，流程：
```
VISA配置串口（串口号/波特率9600/数据位8/停止位1/无校验）
  → VISA写入（发送命令字符串 + 换行符）
  → VISA读取（接收 Arduino 返回的数据）
  → 显示到前面板
```

- NI-VISA 19.0 通过 NI Package Manager 安装
- 参考文档：`说明文档/通信系统综合设计与实践资料下发/2.LaLabVIEW串口通信/`
